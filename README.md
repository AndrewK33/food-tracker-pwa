# Дневник питания — PWA для телефона

Это простое приложение без регистрации для учета калорий и БЖУ.

## Что умеет

- сканирование штрихкода камерой;
- поиск продукта через Open Food Facts;
- добавление продукта по фото;
- повторное фото, возврат назад и добавление 2–3 фото с разных ракурсов;
- локальное распознавание фото прямо в браузере через TensorFlow.js + MobileNet;
- опциональное более точное распознавание через OpenAI API, если пользователь сам указал свой API-ключ;
- ручное редактирование БЖУ;
- округление КБЖУ до одного знака после запятой;
- автоматический пересчет КБЖУ при изменении веса порции;
- дневная таблица питания;
- цели по калориям, белкам, жирам и углеводам;
- история по дням;
- график калорий и БЖУ за последние 14 дней;
- переход к графику по нажатию на карточку калорий;
- избранные продукты;
- копирование вчерашнего дня;
- учет веса и талии;
- экспорт и импорт JSON;
- локальное хранение данных на телефоне.

## Важно про фото

Распознавание по фото — это подсказка, а не медицинский или лабораторный расчет. Приложение не определяет вес по фото. После распознавания нужно выбрать продукт, проверить КБЖУ и указать вес вручную. Если результат не подходит, можно вернуться назад, сделать новое фото или добавить второе фото с другого ракурса.

По умолчанию используется локальная модель MobileNet:

- работает без API-ключей;
- не отправляет фото на сервер приложения;
- работает прямо на устройстве;
- может ошибаться и знает ограниченный набор классов ImageNet.

Если нужен более точный режим, откройте `⚙️ Настройки`, включите OpenAI и вставьте свой API-ключ. Ключ сохраняется только в `localStorage` этого браузера.

### Несколько фото

Второе фото может помочь, если на первом плохой свет, продукт закрыт рукой/упаковкой или виден только один ракурс. В приложении можно добавить до 3 фото за один раз.

Важно: несколько фото не гарантируют правильный ответ. Они просто дают модели больше контекста. В OpenAI-режиме несколько фото могут стоить дороже, потому что обрабатывается больше изображений.

## Риски OpenAI-режима

Проект рассчитан на GitHub Pages, поэтому backend отсутствует. Это значит:

- общий API-ключ нельзя вшивать в код и коммитить в репозиторий;
- каждый пользователь должен использовать свой ключ;
- ключ, введенный в браузере, хранится локально на устройстве, но frontend-среда не считается идеально защищенной;
- прямой запрос из браузера может перестать работать, если у API возникнут CORS-ограничения;
- при ошибке OpenAI приложение автоматически попробует локальную модель.

Для production-приложения с общим ключом нужен backend/proxy. В этом проекте backend сознательно не добавлен, чтобы приложение оставалось максимально простым и запускалось на GitHub Pages.

## Источники БЖУ

Приложение использует Open Food Facts для поиска продуктов по штрихкоду и названию. Данные открытые, но могут быть неполными или ошибочными, поэтому в приложении есть ручное редактирование.

## Конфиденциальность

Дневник, цели, избранное, вес, талия и настройки хранятся локально в `localStorage` браузера на устройстве. Регистрации и сервера приложения нет.

Фото в локальном режиме обрабатывается на устройстве. Фото в OpenAI-режиме отправляется напрямую в OpenAI API с ключом пользователя.

## Инструкция для пользователя: как добавить свой OpenAI API-ключ

В приложении есть встроенная помощь. Нажмите `?` вверху экрана или `?` рядом с блоком `Распознавание по фото` в настройках.

Простыми словами:

1. Откройте приложение и нажмите `?`.
2. Выберите `Добавить OpenAI API-ключ`.
3. Нажмите кнопку `Открыть platform.openai.com/api-keys`.
4. Войдите в аккаунт OpenAI или зарегистрируйтесь.
5. Нажмите `Create new secret key`.
6. Скопируйте ключ сразу после создания.
7. Вернитесь в приложение.
8. Откройте `⚙️ Настройки`.
9. Включите `Использовать OpenAI, если ключ указан`.
10. Вставьте ключ в поле `Ключ OpenAI`.
11. Нажмите `Сохранить настройки`.
12. Теперь кнопка `📸 По фото` сначала попробует OpenAI, а если не получится — вернётся к локальной модели.
